Hybrid Search Implementation

retriever_service.py
def get_relevant_context(query, top_k=5):
    # Generate sparse and dense vectors
    sparse_vec = splade.encode(query)
    dense_vec = openai.embed(query)

    # Perform hybrid search in Pinecone
    results = index.query(
        vector=dense_vec,
        sparse_vector=sparse_vec,
        top_k=top_k,
        include_metadata=True
    )

    # Format for LLM Context Window
    context_str = "\n".join([
        "Date: {0}, Close: {1}".format(
            m['date'], m['close']
        ) for m in results.matches
    ])
    
    return context_str

Workflow Breakdown

01

Data Ingestion

Real-time websocket connection to AlphaVantage streams OHLCV data. Incoming streams are buffered and normalized for vectorization.

02

Hybrid Search Retrieval

Queries are executed using both dense vectors (semantic similarity) and sparse vectors (keyword matching) to find historically similar market conditions.

03

Context Window Optimization

Retrieved financial documents are re-ranked and compressed to fit within the GPT-4 context window, maximizing information density for the model.

04

Forecasting & Output

The LLM generates a structured JSON forecast including confidence intervals and potential market drivers based on the provided context.
