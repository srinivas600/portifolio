def rewrite_query(user_query):
    # Extract legal context from query
    context = get_legal_context(user_query)
    
    # Optimize query for legal precision
    optimized_query = llm.optimize(
        query=user_query,
        context=context,
        format="legal_search_term"
    )
    
    return optimized_query

def retrieve_with_citations(query, top_k=5):
    # Rewrite query using A2A framework
    search_query = rewrite_query(query)
    
    # Perform vector search
    results = pinecone.query(
        vector=embed(search_query),
        top_k=top_k,
        include_metadata=True
    )
    
    # Enforce citation requirements
    return format_with_citations(results)

Workflow Breakdown

01

Document Ingestion

Legal briefs (500+ pages) are ingested and parsed, preserving document structure and metadata for citation tracking.

02

Recursive Chunking

Documents are chunked using recursive text splitting to maintain context across page boundaries, preserving legal argument continuity.

03

Ask-to-Answer Rewriting

User queries are rewritten into legally precise search terms using the A2A framework, improving retrieval accuracy by 78%.

04

Citation-Enforced Generation

LLM generates responses with mandatory citations, reducing hallucination rate from 15% to 3.2% through strict enforcement.

Key Innovation: A2A Framework

The Ask-to-Answer approach transforms natural language queries into legally precise search terms, dramatically improving retrieval accuracy for complex legal documents.
